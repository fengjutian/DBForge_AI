**Electron 桌面数据库管理工具（AI 辅助 SQL）需求文档（SRS）**

**文档版本**：v1.1  
**编写日期**：2026年4月  
**项目名称**：DBForge AI（暂定名，可后续修改）  
**文档目的**：定义 Electron 桌面应用的核心功能、架构、非功能性要求，为后续设计、开发、测试提供完整依据。

### 1. 项目概述

#### 1.1 项目背景
开发一款跨平台（Windows / macOS / Linux）桌面数据库管理工具，核心目标是帮助开发者、DBA、数据分析师**高效连接 MySQL 数据库**，通过**自然语言 + AI** 快速生成 SQL，并提供专业级 SQL 编辑与执行体验。同时支持**数据库备份**功能，满足日常运维需求。

#### 1.2 核心价值
- 降低 SQL 编写门槛（自然语言 → SQL）
- 提供 VS Code 级别的 SQL 编辑体验
- 保证数据库操作安全（AI 生成 SQL 必须人工确认 + 危险操作拦截）
- 一键备份还原，保护数据资产
- 完全本地化运行，无需服务器部署

#### 1.3 项目范围
**本次版本（v1.0）范围**：
- MySQL 数据库连接与管理（优先支持）
- AI Text-to-SQL（自然语言生成 SQL）
- Monaco Editor 专业 SQL 编辑器
- SQL 执行与结果展示
- 数据库逻辑备份（mysqldump）

**不在本次范围**（可作为 v2.0 规划）：
- PostgreSQL / SQLite / Oracle 等多数据库支持
- AI Agent 自动执行 SQL（需人工确认）
- 数据库设计 / ER 图 / 迁移工具
- 云端同步 / 团队协作

### 2. 目标用户与角色

| 用户角色       | 主要需求                                   | 优先级 |
|----------------|--------------------------------------------|--------|
| 开发者         | 快速写查询、AI 辅助生成 SQL、调试         | 高     |
| DBA / 运维     | 数据库连接管理、备份还原、Schema 查看     | 高     |
| 数据分析师     | 自然语言查询、结果分析、导出               | 中     |
| 学生 / 新手    | 学习 SQL、AI 解释结果                      | 中     |

### 3. 功能需求

#### 3.1 数据库连接管理
- 支持 MySQL 连接配置（主机、端口、用户、密码、数据库）
- 连接池管理（主进程实现）
- 测试连接功能
- 保存连接配置（electron-store，密码加密存储）
- 左侧连接列表支持多个连接同时保持活跃，每个连接拥有独立的 SQL 编辑器 Tab
- 连接状态实时显示（在线 / 断线 / 连接中），有明确视觉标识（颜色 + 图标）
- 断线自动重连提示
- 连接配置支持导入 / 导出（密码字段脱敏），方便团队共享

#### 3.2 Schema 浏览器
- 自动获取数据库表结构（表名、字段、类型、主键、外键）
- 树形结构展示（数据库 → 表 → 字段）
- 一键刷新 Schema（供 AI 使用）
- 右键表名可快速预览表数据（查询前 100 行，无需手写 SQL）

#### 3.3 AI Text-to-SQL 模块
- 用户输入自然语言问题（如"查询 2025 年销售额前 10 的产品"）
- 自动获取最新 Schema 传入 Prompt
- 调用 LLM（支持 OpenAI / Groq / Claude / DeepSeek / Ollama 等）生成 SQL
- **AI 模式**：
  - **只读模式**（默认）：仅生成 SELECT 语句，适合数据分析师 / 新手
  - **完整模式**：允许生成 INSERT / UPDATE / DELETE / ALTER 等语句，适合开发者 / DBA，需在设置中手动开启
- **安全机制**：
  - 危险操作（DROP / TRUNCATE / DELETE without WHERE）在编辑器中高亮警告，执行前弹出二次确认弹窗
  - 生成后必须在 Monaco 编辑器中展示，用户**手动确认**后才能执行
- 生成结果同时返回自然语言解释
- 支持 Few-shot Prompt + JSON Mode 控制输出

#### 3.4 SQL 编辑器（Monaco Editor）
- 集成 `@monaco-editor/react`
- 支持 MySQL 语法高亮、自动补全（表名、字段名基于 Schema 动态补全）
- SQL 格式化、查找替换、主题切换（跟随系统暗黑模式）
- 支持只读模式（AI 生成后可编辑）
- 错误实时提示（可选集成 SQL 校验）
- SQL 片段收藏（Snippets）：支持保存常用 SQL 片段，快速插入编辑器

#### 3.5 查询执行与结果展示
- 执行 SQL（主进程安全执行）
- 结果以表格形式展示（支持排序、筛选、导出 CSV / JSON / Excel）
- 分页加载（大结果集）
- AI 结果总结：用户主动点击"AI 解释"按钮触发，对执行结果进行自然语言总结与洞察（避免自动触发产生额外 API 费用）
- **查询历史记录**：
  - 本地保存，字段包含：SQL 内容、执行时间、耗时、返回行数、所属连接名
  - 支持关键词搜索 / 按连接过滤历史
  - 支持一键重放历史 SQL
  - 历史记录上限可配置（默认 1000 条，防止 SQLite 无限增长）

#### 3.6 数据库备份与恢复
- 一键备份当前数据库（使用 `mysqldump`）
  - 支持 `--single-transaction`、`--routines`、`--triggers` 等参数
  - 用户选择保存路径（默认下载目录，文件名带时间戳）
  - 备份后显示文件大小与耗时
- 支持压缩（.sql.gz 可选）
- 恢复功能（使用 `mysql` 命令导入备份文件）
- 备份前 Schema 与数据量预估提示
- 备份路径记录与快速打开所在文件夹

#### 3.7 系统设置
- 全局配置：AI 提供商 API Key、模型选择、温度
- AI 模式切换：只读模式 / 完整模式
- **mysqldump 路径配置**：
  - 应用启动时自动探测常见路径（`/usr/bin/mysqldump`、`C:\Program Files\MySQL\...` 等）
  - 探测失败时引导用户手动指定，并提供"检测"按钮验证路径有效性
- 主题、语言（中文优先）、自动备份开关
- 查询历史记录上限配置
- 日志查看

### 4. 非功能需求

#### 4.1 性能要求
- 应用启动时间 ≤ 2 秒（基准：8GB RAM + SSD 机器）
- AI 生成 SQL 响应时间 ≤ 3 秒（本地 Ollama 可接受 5 秒）
- 大结果集（10万行）加载不卡顿（分页 + 虚拟列表）
- 备份过程不阻塞主界面（进度条 + 子进程）

#### 4.2 安全要求
- 启用 Electron `contextIsolation` + `nodeIntegration: false` + Sandbox
- 所有数据库操作在 Main Process 执行
- SQL 执行前二次校验危险关键词
- API Key 加密存储，不明文出现在日志
- 备份文件默认不包含敏感信息（用户可选择）

#### 4.3 错误处理规范
- 连接失败：显示具体错误码 + 常见原因说明 + 排查建议
- AI 调用失败：提示错误原因，支持一键重试或切换备用模型
- 备份失败：保留完整错误日志，提示用户查看日志并给出可能原因
- SQL 执行失败：高亮错误行，展示 MySQL 原始错误信息 + 友好说明

#### 4.4 可用性与兼容性
- 支持 Windows 10+、macOS 11+、Linux（Ubuntu / Fedora）
- 最低硬件要求：8GB RAM、SSD 存储
- 中文界面优先，支持英文切换
- 键盘快捷键丰富（Ctrl + Enter 执行、Ctrl + K 格式化等）
- 无障碍支持（基础）

#### 4.5 可维护性
- 使用 TypeScript
- 模块化架构（Main / Renderer 分离）
- IPC 接口清晰定义
- 完整错误处理与用户友好提示

### 5. 技术栈要求

| 层级         | 技术选型                                          |
|--------------|---------------------------------------------------|
| Electron     | Electron 最新版 + Vite + React + TS              |
| UI           | Shadcn/ui + Tailwind CSS                         |
| 编辑器       | @monaco-editor/react                             |
| 数据库       | mysql2（连接池）                                 |
| AI           | LangChain.js + OpenAI / Groq / Ollama            |
| 存储         | electron-store（配置） + better-sqlite3（本地历史） |
| 导出         | exceljs（Excel 导出）                            |
| 备份         | child_process + mysqldump                        |
| 打包         | electron-builder                                 |

### 6. 数据流（核心流程）

1. **AI 查询流程**：用户输入自然语言 → IPC → Main 获取 Schema → 调用 LLM → 返回 SQL → Renderer 渲染到 Monaco 编辑器 → 用户确认 → IPC 执行 → 返回结果（用户可手动触发 AI 总结）
2. **备份流程**：用户点击备份 → Main 自动探测 mysqldump 路径 → 弹出保存对话框 → spawn mysqldump → 写入文件 → 返回结果

### 7. 界面布局建议（主界面）
- 左侧：连接列表（含状态标识）+ Schema 树
- 中间：Monaco SQL 编辑器（可多 Tab，每个连接独立 Tab）
- 右侧/下方：自然语言输入 + AI 生成按钮 + 执行结果表格 + AI 解释面板（手动触发）

### 8. 验收标准
- 所有功能点均可通过 UI 操作完成
- AI 生成的 SQL 准确率 ≥ 85%（常见查询场景）
- 备份文件可在 MySQL Workbench 中完整恢复
- 安全机制能拦截所有危险 SQL（只读模式下不生成写操作）
- 跨平台打包后可正常运行
- mysqldump 路径自动探测成功率 ≥ 90%（主流安装路径）
- 错误场景均有明确提示，无白屏或无响应情况

### 9. 假设与约束
- 用户本地已安装 MySQL 客户端（mysqldump / mysql）
- 用户拥有合法 LLM API Key 或可使用本地 Ollama
- 数据库权限充足（SELECT、SHOW 等）
- 网络环境允许访问 LLM 服务（离线模式可选）
- 最低运行环境：8GB RAM、SSD、64 位操作系统
